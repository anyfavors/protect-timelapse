## Project

**Protect Timelapse** — a self-hosted FastAPI service that connects to a UniFi Protect NVR, captures camera snapshots at configurable intervals, and assembles them into MP4 timelapse videos. Managed through an Alpine.js + Tailwind CSS SPA. Full spec: [project-design.md](project-design.md).

## Architecture

Three async background workers run inside a single Uvicorn process, managed by the FastAPI `lifespan` context in `app/__init__.py`:

1. **Capture Worker** (`app/capture.py`) — `APScheduler AsyncIOScheduler`. Per-project `IntervalTrigger`. Before saving a frame: runs astronomical check (`astral`), then luminance check (`Pillow`). Writes JPEG to `/data/frames/{project_id}/{timestamp_utc}.jpg` and thumbnail to `/data/thumbs/{project_id}/{timestamp_utc}.jpg` (320px, quality=60).
2. **Render Worker** (`app/render.py`) — strictly sequential queue (one concurrent `ffmpeg` job). Picks `status='pending'` rows from `renders` table. Supports deflicker, timestamp burn-in (`drawtext`), frame skipping (`select` filter), and concatenation rollups (`-f concat -c copy`). Uses `asyncio.create_subprocess_exec`, capped at `-threads 4`.
3. **Maintenance Worker** (`app/maintenance.py`) — daily cron. Enforces `retention_days`, prunes renders, checks disk usage. If free space < `disk_warning_threshold_gb`, pauses all active projects (`status='paused_error'`) and fires webhook.

WebSocket endpoint `WS /api/ws` broadcasts real-time events (`capture_event`, `render_progress`, `render_complete`, `disk_update`, `notification`) to all connected clients via `app/websocket.py`.

## Database

SQLite3, WAL mode, no ORM. Every connection from `get_connection()` must immediately execute:

```sql
PRAGMA foreign_keys = ON;
PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
```

Tables: `settings` (single row, id=1), `projects`, `frames`, `renders`, `project_templates`, `notifications`. Schema versioned via `PRAGMA user_version`; migrations applied sequentially in `init_database()`.

Key invariants:
- All timestamps stored and compared in **UTC**. Local time (`TZ`) only at display layer and for `astral` sunrise/sunset math.
- `projects.frame_count` is a denormalized counter (incremented by capture worker) — never use `COUNT(*)` for dashboard polling.
- On `DELETE /api/projects/{id}`: run `shutil.rmtree()` for frames/thumbs/renders directories **before** the SQL delete (so retries are safe if SQL fails).
- `ON DELETE CASCADE` handles `frames`, `renders`, `notifications` rows automatically.

## Key module responsibilities

| File | Responsibility |
|------|---------------|
| `app/config.py` | Pydantic `BaseSettings` — all env-var reads |
| `app/database.py` | `get_connection()`, `init_database()`, migration runner |
| `app/protect.py` | `ProtectClientManager` singleton with `asyncio.Lock()`, lazy init, auto-reconnect |
| `app/capture.py` | `APScheduler` instance, `snapshot_worker()`, visibility filtering pipeline |
| `app/render.py` | Sequential render queue, all `ffmpeg` subprocess calls |
| `app/thumbnails.py` | Pillow thumbnail generation (320px wide, quality=60) |
| `app/notifications.py` | Writes to `notifications` table AND fires external webhook via `httpx` |
| `app/websocket.py` | `ConnectionManager`, `broadcast()` function |

Routes under `app/routes/`: `health`, `projects`, `frames`, `renders`, `cameras`, `settings`, `templates`, `notifications`.

## uiprotect API usage

```python
from uiprotect import ProtectApiClient

# List cameras (after client.update()):
for camera in client.bootstrap.cameras.values():
    print(camera.id, camera.name)

# Snapshot (returns JPEG bytes):
snapshot: bytes = await camera.get_snapshot()
snapshot: bytes = await camera.get_snapshot(width=1920, height=1080)

# Historical video chunk (for historical extraction):
chunk: bytes = await camera.get_video(start_dt, end_dt)
```

Historical extraction: chunk into 1-hour segments → `ffmpeg` extract frames at `fps=1/{interval_seconds}` → rename files to UTC timestamps → generate thumbnails → set `status='completed'`.

## Frontend

Single HTML shell (`templates/index.html`) + `static/app.js`. Alpine.js 3 SPA with tab-based views: Dashboard, Project Detail, Create/Edit, Render, Settings.

Tailwind v4: multi-stage Docker build (`node:22-slim` stage), `@tailwindcss/cli` via npm. Config in `app.css.src`. Dark mode via `@custom-variant dark (&:where(.dark, .dark *))`.

## Tests

`tests/conftest.py` provides:
- `tmp_db` fixture — temporary SQLite file with `init_database()` applied
- `client` fixture — FastAPI `TestClient` with lifespan stubbed (no real NVR, no workers)

All NVR and worker interactions are replaced via `monkeypatch.setattr`.
